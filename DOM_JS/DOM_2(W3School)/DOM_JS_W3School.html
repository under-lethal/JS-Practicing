11. Event Bubbling OR Event Capturing

There are two ways of EVENT propagation:
bubbling and capturing

.

Event PROPAGATION is a way of DEFINING the element OREDER - when an EVENT occurs.
If you have a "< p>" element INSIDE a "< div>" element (and the user CLICKS on the "p" element) - which element's "clicks" event should be handled first?

In bubbling - the inner most element's event - is HANDLED first - and then the outer.
*the < p> element's CLICK EVENT is HANDLED first, then the < div> - element's CLICK EVENT.

IN capturing the outer most element's event - is HANDLED first - and the INNER.
*the < div> element's CLICK EVENT will be HANDLED first, then the < p > element's click event.

12. JavaScript - DOM Navigation

With the HTML DOM - you can NAVIGATE the NODE TREE using node relationships.
Everything in a HTML document is a node.

DOM Nodes

  • The entire document - is a document node.
  • Every HTML element - is a element node.
  • The TEXT inside HTML elements - are text nodes.
  • Every HTML attribute - is a attr node.
  • New nodes - can be created - and all nodes can be mdified or deleted.

Node Relationships

The nodes - (in a node Tree) - have a hierarchical relationship - to each other.
The terms parent, child and sibling are used to describe the relationships.

In a node tree - the TOP node - is called the ROOT(or root node).
Every node - has exaclty - one parent - except the root (which has no parent).
A node - can have a number of children.
Siblings (brothers or sisters) - are NODES - with the SAME Parent.

  • parentNode
  • childNodes
  • firstChild
  • lastChild
  • nextSibling
  • previousSibling

The Difference Between an HTMLCollection and a NodeList

A NodeList and an HTMLcollection is very much the same thing.
Both are array-like collections/(list) of nodes/(elements) - extracted from a document.
The nodes can be accessed by INDEX NUMBERS. Both have a length property - that RETURNS a number of elements in the LIST/(collection).

An HTMLCollection is a collection of - documents ELEMENTS.
A NodeList is a collection of documents nodes.(ELEMENT nodes, ATTRIBUTES nodes and TEXT nodes).

HTMLCollections ITEMS can be ACCESSED by their name, id, or index number.
NodeList ITEMS can ONLY be accessed by their index number.

An HTMLCollection is always a live collection
Example: If you ADD a < li > element - to a list in the DOM - the list in the HTMLCollection - will also change.

A NodeList is most often a static collection
Example: If you add a "< li >" element to a list in the DOM - the list in NodeList will not change.

The getElementByClassName() and getElementByTagName() METHODS - return a live HTMLCollection.
The querrySelectorAll() METHOD returns a static NodeList.
The childNodes PROPERTY - returns a live NodeList.
