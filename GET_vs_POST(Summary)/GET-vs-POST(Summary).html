GET vs POST - Summary

GET

BODY: No

  • When you try to SEND Data - you're LIMITED - to how much DATA you can SEND - to the SERVER.
  • The ONLY way (that you can send DATA) - is in the URL.
  • If you wanna send a REQUEST - you're querrying(interoghezi) - the SERVER (ex: google.com) - and Google - start adding these URL PARAMETERS.("/html/default.asp")

DATA Request LIMIT:

  • The URL - is the only PLACE you can send Data through.
  • If you wanna send an info - it goes through the URL (because - GET - don't have a BODY.)
  • Because GET - uses an URL - there is a LIMIT in the URL length. (2048 bytes).

DATA TYPE:

  • Because GET - uses an URL -> (ASCII only).
  • In your URL - you cannot put UTF-8 characters.
  • Because you can ONLY send ASCII characters - they're LIMITED to what kind of DATA you can SEND.

SAFE:

YES

It's not reffering - to UNSECURE.
SAFE - is DEFINITION of a safe HTTP Method.

  • Is a METHOD - that does NOT change The Resource that is Requested.
  • For example: If I do: www.google.com - i'm not changing that Resource.
  • If I am REQUESTING - my user FB/Instagram Profile - i'm NOT chaning ANYTHING / I didn't UPDATE my name.
  • I'm just asking: "Hey give me my Profile.

  • Only READS

CACHING and PreFETCHING

YES

  • Caching - is the ABILITY of the BROWSER - to KEEP Responses - that COMMING back from the SERVER - WITHOUT making ANOTHER request.
  • GET is perfect for CACHING and PreFETCHING - because - if you make a REQUEST - to see my Profile (Fb/Instagram) - I can CACHE that.

BOOKMARKable / History

YES

  • All the GET Request - are SEND through the URL - (so - You have LONG URL - with a LONG set of PARAMETERS.
  • If you BOOKMARK that URL - AND VISIT LATER - you get the same PAGE.
  • Because: GET - has everything in the URL - nothing in the BODY.
  • Everyone - SEES that URL.
  • When you do this www.google.com - is ALWAYS a GET Request.
  • In Almost cases USERS/CLIENTS - make (most of the time) - GET Request.
  • Click on a Link - it's a GET Request.
  • Visit a Page - it's a GET Request.

SECURITY

NO

Neither - are (so) SECURE - but GET is less SECURE than POST.

  • Your password will be VISIBLE in the URL.
  • If you're sending a REQUEST - with your poassword in it - it's gonna show up in the URL.

  • It's gonna be added to BROWSER History.
