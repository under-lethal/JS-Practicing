I.Regular Expression

A Regular Expression - is a SEQUENCE of characters - that forms a SEARCH PATTERN.
A Regular Expression - can be a single character or more complicated PATTERN.
When you search for a DATA in a TEXT - you can use SEARCH PATTERN to describe - what are you SEARCHING for.
A Regular Expression can be used to PERFORM all types of text search and text replace. 

		/w3Schools/i - is a REGULAR EXPRESSION.
		/w3Schools/ - is a PATERN. (to be USED in SEARCH)
		i - is a MODIFIER.(MODIFIES the SEARCH to be case-insensitive).

2.Using String Methods

In JS - Regula Expressions - are used with the TWO STRING METHODS. search() and replace().
search() - uses an EXPRESSION to search - for a MATCH - and RETURN the POSITION of the MATCH.
replace() - returns a MODIFIED STRING - where the PATTERN is REPLACED.

The ignoreCase property - specifies WHETHER or NOT the "i" modifier is set.

	Note: exec( ) and test( ) reset lastIndex to 0 if they do not get a match.

Flags

Flags are a way of modifying the BEHAVIOUR of a RegExp

/g

/g - standing for GLOBAL. That means - don't JUST MATCH The First Thing - MATCH All of Them
/i - for Case In Sensitive. 

		< script type="text/javascript">
		var s = "unicorns and rainbows and cupcakes";
		var r = /[a-z]+/g; // <- NOtice: the "FLAG" goes after the REGULAR Expression.
		s.match(r); //["unicorns", "and", "rainbows", "and", "cupcakes"]; - we get an array WITH EVERY SINGLE MATCH of THAT RegExp.

Let's capitalize "And" and "Cupcakes"

		var s = "unicorns and rainbows And Cupcakes";
		var r = /\b[a-z]+\b/g; 
		s.match(r); 	// [unicorns and rainbows] not "And Cupcakes" - because I'M ONLY LOOK FOR LOWERCASE LETTERS.

		2. Add another Flag "\i".
		var r = /\b[a-z]+\b/gi;
		s.match(r);		//["unicorns and rainbows not "And Cupcakes"];

		This is how The Flags work:
		a. I need Global Flag - to match the RITGH Expression MULTIPLE TIMES in the string
		b. "i" Flag - is for CASE SENSITIVITY.
