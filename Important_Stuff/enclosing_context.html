We have a function called "outerFunction()".
That Function is returning another Function() - inside of it called innerFunction();
Here: we calling outerFunction(), with the variable 'outside'.
Then we calling that newFunction('inside') - with the variable inside.

When we calling "outerFunction() - we have this ' + outerVariable' - which we SET to outside ~ outerFunction(outerVariable). Then we have this "innerFunction()"

The reason we're able to acces this (outerVariable) inside of the outerFunction(outerVariable) - is because of CLOSURES.
How is the innerFuncton() - able to ACCESS this (outerVariable) - even after is done being executed...?
outerVariable is OUT of SCOPE. Q&A: Because of the closures

When most people think of closures - they think of functions - inside other functions.

In JS anything on the inside - has acces to the things on the outside of its scope.
Essentially - it has acces to it's parent scope, its parent, parent scope and so on...
