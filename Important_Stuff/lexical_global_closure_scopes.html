Lexical Scope - defines how VARIABLES names are RESOLVED in NESTED Functions().

If we have a CHILD Function() (within a PARENT Function() - the CHILD Function has ACCES to the SCOPE of the PARENT Function() - and HAS ACCES to the GLOBAL Scope.

Nested Child Functions - have acces to the SCOPE of THEIR Parent Functions().

THIS IS CONFUSED WITH "Closure".

Let's display CLOSURE.

A CLOSURE - is a Function having acces to the PARENT Scope - even after the PARENT Function() has closed.
A CLOSURE is created WHEN we define a Function - NOT WHEN A Function is EXECUTED.(...)
	CLOSURE - IS...AFTER WE SET THE boundaries (like{...}) - we're TELLING the childFunction what is has CLOSUER OVER(at the time it is defined).
	It is DEFINED within the parent Function() - and that GIVING (the childFunction) - acces to not only IT'S LOCAL SCOPE,
	but also have ACCESS to the parentFunction() SCOPE - and it has access to the GLOBAL Scope.
	It could be sait - IT HAS CLOSURE - over all 3.

Now to have access to the parent SCOPE - even after the PARENT Function() has CLOSED.

Instead of CALLING the "chidlFunction()"(INSIDE the parentFunction()) - WE actually need to "return" - the childFunction().
