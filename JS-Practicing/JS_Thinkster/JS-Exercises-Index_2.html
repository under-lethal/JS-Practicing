"for...in" in an Array

for of - will not work on a "person object" - because person is not ITERABLE.
The "for...in" - loops through INNUMERABE (care nu se pot numara) PROPERTIES - while "for...of" loops through ITERABLE (*reprezentare a unor serii de elemente care pot fi repetate) OBJECTS such as (arrays, sets, map).

The for ... in loop is very similar to the for ... of loop. The only difference is that instead of giving you the item in the array, it gives you the index each time through the loop.
As you can see, we don't need to use index notation to get the item in the array; it's just given to us each time through the array.
When you use a for ... of loop, you get access to each element in the array, one each time through the loop. When you use a for ... in loop on an array, however, you get access to the index of each item in the array.

while, do...while

The while and do ... while loops are very similar. They both allow you to run a certain piece of code over and over again until some condition is met.
That condition goes inside parentheses after the while keyword.

The condition, can be anything that you want, as long as it evaluates truthy (if you want to go in to the loop) and will eventually evaulate to falsy.
If it never evaluates to falsy, you'll be stuck in an infinite loop, or a you'll be stuck in the loop with no way to exit.
So make sure that you provide some sort of expression that can evaluate to both truthy and falsy at least once.

In this loop, you provide a condition that, until it evaluates as false, will continue running. In this case, you don't need an iterable to loop over. 

	while(expressionIsTrue){// Do whatever you want}

	The do ... while loop is almost the exact same. The only difference is the syntax of how it's started.

	do {//Do whatever you want} while(expression is TRUE)

In this example, the name variable in the sayHello function is the parameter. When the function is called at the end of the example, 'Preston' is the argument.

Named Functions()

Named functions in JavaScript is just a fancy way to refer to a function that uses the function keyword and then a name you can use as a reference to that function. Here's an example:

	function myFunction(someParameter) {
    // Do something
	}

A named function has 4 basic parts: the function keyword, the name you give it and use to refer to it, the parameter list, and the body of the function.

NAMED Functions can also be assigned to ATTRIBUTES on objects - and then you CALL the Function - on the OBJECT
	function sayHi(){
		console.log(`Hi my name is ${this.name}.`);
	}

	const person = {
		name: "Preston Lamb",
		sayHi
	}
	person.sayHi();

NAMED Functions are - one of the most common ways that functions are called in JS.

One other thing that is important to mention - is HOW TO Return a VALUE from a FUNCTION - using the [return] KEYWORD.

	function substract(sum1,sum2){
		return sum1 - sum2;
	}

	Function takes two parameters [mu1,num2] - and RETURNS the value of "num1"- "num2".
	That RETURNED VALUE - can be assigned to a VARIABLE, 
	or LOGGED to the console,
	or PASSED as the PARAMETER to another Function.

Rest Parameters

"Rest" parameters in functions take all the arguments (passed into a function) (starting from a gien point) - and put them into single parameter - inside the function - as an array.

< p id="rp">< /p>
< script type="text/javascript">
	function addNumbers(name, ...numbers){
		let sum = 0;
		for (const num of numbers){
			sum += num;
		}
		return 'Total of ' + name + ': ' + sum;
	}
	const totalPrice = addNumbers('Price', 1,2,3,4,5);
	 document.getElementById('rp').innerHTML = totalPrice;
< /script>


	The numbers 1,2,3,4,5 - are ALL ADDED into the "numbers" parameter as the rest parameter.
	Once you've declared a rest parameter you can't decalre any other parameters, (including_ other rest parameters, after it.
What is allows you to do is - to name a certain number of parameters to the function - and then catch the rest of them in an array.
< p id="rp1">

< script type="text/javascript">
	function addition(setName, ...numbers){
		const total = numbers.reduce((prev, next) => (prev += next), 0);
		return `Total of ${setName}: ${total}`;
	}
	const result1 = addition('years', 1,2,3,4);
	document.getElementById('rp1').innerHTML = result1;
< /script>

This function takes a parameter called setName.
Then, we capture the rest of the arguments passed to the function in the numbersarray.
That arguments includes all the arguments or parameters passed to a function.
The rest parameter - only includes all the parameters that weren't named.
The rest parameter - is actually an array.
Arguments (array-like) - cannot have methods (for Each, .reduce) on arguments.
